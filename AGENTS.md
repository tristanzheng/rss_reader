1. PLAN first, then clear context and execute.
2. Always ask for clarification if the question is ambiguous.
3. Do TDD (Test-Driven Development) to ensure code quality and correctness, writing tests before implementing functionality.
4. For each task, break it down into smaller, manageable steps and execute them sequentially, use a markdown file to keep track of the steps and progress.
5. All markdown files should be named with a clear and descriptive title, and should be organized in a logical folder structure for easy navigation and reference.
6. Regularly review and update the markdown files to reflect any changes or new information, ensuring that they remain accurate and relevant.
7. Use version control (e.g., Git) to track changes to the markdown files and code, allowing for easy collaboration and rollback if necessary.
8. When executing code, ensure that it is well-documented and follows best practices for readability and maintainability, including clear variable names, comments, and consistent formatting.
